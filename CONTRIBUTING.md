# Contributing to `iso-management-wisdom`

Thank you for contributing. This repository welcomes improvements, but it also needs consistency and traceability because it represents ISO-aligned management guidance.

## What to Contribute

Good contribution types:

- improvement proposals for existing guidance
- new role coverage
- ISO revision follow-ups
- translation and terminology fixes
- harness-engineering usage examples
- documentation and template improvements

## Recommended Entry Points

- `good first issue`: small docs, translation, wording, and structure fixes
- `help wanted`: broader updates that need outside help
- Issue Forms: structured proposals and corrections
- Discussions: idea triage and open design topics once enabled in repository settings

If Discussions are not yet enabled, open an Issue instead.

## Quality Expectations

Please keep these aligned when relevant:

- `README.md`
- `docs/`
- `examples/`
- `skill/.../SKILL.md`
- `skill/.../references/role-index.md`

For substantive changes, explain:

- what changed
- why it changed
- which roles or ISO references are affected
- whether Japanese and English remain aligned
- whether examples or harness docs also need updates

## Language Parity

The repository is bilingual. If a change affects public guidance, contributors should update both Japanese and English unless the change is explicitly scoped to one language.

## ISO and Evidence Expectations

This repository does not reproduce ISO standards. Contributions should avoid copying proprietary standard text and should instead provide:

- references
- checklists
- interpretation guidance
- examples
- process design suggestions

When adding or revising guidance, include the basis for the change as clearly as possible.

## Pull Requests

Use the PR template and make the review easy:

- summarize the change clearly
- describe the rationale
- mention impacted roles or standards
- call out language-parity status
- note harness-engineering impact if applicable

## Good First Contributions

Examples of contribution ideas that fit well:

- improve a README example
- tighten wording in `AGENTS.md.example`
- add a missing keyword to `role-index.md`
- clarify how a role should be selected
- improve bilingual consistency

## Repository Settings

Some community features, such as Discussions and labels, depend on GitHub repository settings. If they are not enabled yet, the docs in this repository should be treated as the intended operating model.
