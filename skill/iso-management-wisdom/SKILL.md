# ISO Management Wisdom

## 目的
ISO規格は広く一般化された経営の叡智である。
このスキルは、23の組織ロールに蓄積されたISO準拠の知見を、
依頼の文脈に応じて動的に選定・適用し、即座にアクションへ変換する。

## いつ使うか（トリガー）
- 組織運営に関する方針・体制・プロセスの設計や見直し
- 監査・レビュー・経営会議の準備や抜け漏れチェック
- インシデント・事故・不祥事への対応設計
- コードベースやドキュメントのISO観点でのレビュー
- 「何を考慮すべきかわからない」時の網羅的チェック
- セキュリティ/品質/コンプラ/ESG等の専門的判断が必要な時

## 手順

### Step 1: 依頼の解析
ユーザの依頼から以下を特定する:
- **対象領域**: 何についての依頼か（セキュリティ/品質/人事/ESG等）
- **アクション種別**: 何をしたいか（下記5モード）
- **スコープ**: 対象組織/業務/期間
- 不明点があればユーザに質問（最大3問）

### Step 2: ロール自動選定
`references/role-index.md` を参照し、依頼に最適なロール（1〜3個）を選定。
複合的な課題（例: セキュリティ事故 → CISO + Legal + BCM）は複数ロールを組み合わせる。

### Step 3: ロール知見の読み込み
選定したロールの `references/roles/role-*.md` を読み込み、
該当ISO規格の要求事項・チェック項目を取得する。

### Step 4: アクション実行（5モード）

#### Mode A: 回答・助言
「〇〇について教えて」「何を考慮すべき？」
→ ISO規格の要求事項に基づく体系的な回答を提供

#### Mode B: ドキュメント生成
「方針を作りたい」「RACI表が必要」「手順書を書きたい」
→ `references/templates/` のテンプレート + ISO要求事項でドラフト生成

#### Mode C: チェック・レビュー
「現状を点検したい」「監査準備」「コードをISO観点でレビュー」
→ ISO条項ベースのギャップ分析表 or チェック結果を出力

#### Mode D: インシデント対応
「事故が起きた」「インシデント対応」「是正したい」
→ 関連ロール自動組み合わせ + 初動チェックリスト + 報告書ドラフト

#### Mode E: 会議体・プロセス設計
「マネジメントレビューを設計」「定例会議のアジェンダ」
→ ISO条項ベースのアジェンダ/頻度/議事録テンプレート

### Step 5: 検証
- 重要な要求項目が成果物に反映されているか確認
- 抜け漏れがあれば追加ロールの知見で補完
- ユーザにレビューを依頼
