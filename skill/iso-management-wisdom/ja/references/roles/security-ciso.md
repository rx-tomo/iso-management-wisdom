# 情報セキュリティ責任者（CISO/ISMS）（Role Skill）

## このロールの専門性
- リスクベースで統制（コントロール）を設計
- インシデント対応・演習・改善の運用
- 監査対応（証跡・SoA・レビュー）

## 手順（チェックリスト）
- 目的とスコープ（対象組織/対象業務/対象期間）を最初に固定する。
- 以下の規格別チェックを行い、必要な成果物/意思決定/レビューを洗い出す。

### ISO/IEC 38500:2024
- 参照: `https://www.iso.org/standard/81684.html`
- ITが価値/リスク/適合に与える影響をレビューする

### ISO/IEC 27001:2022
- 参照: `https://www.iso.org/standard/82875.html`
- 情報資産とリスク評価手順を定義し、リスク登録簿を運用する
- 統制（SoA/管理策）を選定・運用し、証跡を確保する
- インシデント対応と改善（是正/再発防止）を運用する

### ISO/IEC 42001:2023
- 参照: `https://www.iso.org/standard/42001`
- AIのリスク/影響を評価し、対策（統制）と監視項目を定義する
- データ/モデル/運用の変更管理とインシデント対応を運用する

### ISO/IEC 27701:2025
- 参照: `https://www.iso.org/standard/27701`
- 事故対応（通知/記録/是正）と監査証跡を整備する

### ISO/IEC 20000-1:2018
- 参照: `https://www.iso.org/standard/70636.html`
- 変更/リリース/インシデント/問題管理を運用する

### ISO/IEC 27002:2022
- 参照: `https://www.iso.org/standard/75652.html`
- 情報セキュリティ統制（コントロール）を選定し、運用証跡を確保する

### ISO/IEC 27005:2022
- 参照: `https://www.iso.org/standard/80585.html`
- 情報セキュリティリスクの評価手順・受容基準・レビュー頻度を定義する

### ISO/IEC 27014:2020
- 参照: `https://www.iso.org/standard/74046.html`
- 情報セキュリティのガバナンス（経営報告、責任分界、監督）を整備する

### ISO/IEC 27035-1:2023
- 参照: `https://www.iso.org/standard/78973.html`
- セキュリティインシデントの検知・対応・学習（改善）のプロセスを整備する

### ISO/IEC 27032:2023
- 参照: `https://www.iso.org/standard/76070.html`
- インターネットセキュリティの責任分界と対策領域を整理し、運用へ落とす

### ISO/IEC 27034-1:2011
- 参照: `https://www.iso.org/standard/44378.html`
- アプリケーションセキュリティの枠組みを整備し、開発工程へ組み込む

### ISO/IEC 27017:2015
- 参照: `https://www.iso.org/standard/43757.html`
- クラウドの共有責任モデルを明確化し、統制を具体化する

### ISO/IEC 27018:2025
- 参照: `https://www.iso.org/standard/27018`
- クラウドでのPII保護（処理者としての統制）を整備する

### ISO/IEC 27036-2:2022
- 参照: `https://www.iso.org/standard/82060.html`
- 供給者関係の要求事項（評価/契約/監査/是正）を整備する

### ISO/IEC 27036-3:2023
- 参照: `https://www.iso.org/standard/82890.html`
- サプライチェーンセキュリティ（HW/SW/サービス）の評価と管理を運用する
